# Reflection (Assignment 01)

This project taught me that persona quality is mostly an input design problem, not a UI problem. The chatbot architecture itself was straightforward: a frontend chat window, persona selector, and one API call carrying the selected system prompt. The hard part was writing prompts that produce stable behavior over many user turns. When prompts were vague (for example, just saying "be motivating"), outputs became generic and repetitive. Once I rewrote prompts with detailed persona descriptions, explicit tone instructions, few-shot examples, and response constraints, the conversations became noticeably more consistent and believable.

The GIGO principle became very visible while testing. Low-quality prompt input created shallow output regardless of model strength. High-quality prompt input with concrete communication style, clear boundaries, and examples produced much better answers even on the same model. I also learned that constraints are not optional; they are a safety and quality mechanism. Without constraints, the model can drift into overpromising outcomes or making claims that do not align with responsible mentorship. Adding guardrails such as "never guarantee placements" and "never reveal chain-of-thought" improved reliability.

Implementing persona research pipelines in both JavaScript and Python was useful because it made the process repeatable. Instead of manually collecting notes each time, I can update source URLs and regenerate research snippets from web pages and YouTube transcripts. This creates a cleaner loop for prompt refinement. It also makes documentation easier because prompt choices can be traced back to collected source material.

If I had more time, I would improve three areas. First, I would add stronger retrieval grounding so each persona references verifiable statements from curated source notes. Second, I would run automatic style evaluations (tone consistency, response length, and persona confusion score) across a benchmark set of prompts. Third, I would add production polish: persistent chat history by persona, better loading micro-interactions, and streaming responses. Overall, the biggest lesson is that prompt engineering is product engineering: research quality, structure, and constraints directly shape user trust and output quality.
